# End User Journeys — Phase 0

> 시니어 UX 리서처 점검 결과 (2026-05-20).
> 10개 시나리오를 영화 단위로 분해 + 모호함 90+ 건 결정.
> 코드·UI 구현 시 1차 장관 (canvas) 으로 사용.

## 사용 방법

- **개발자**: 각 시나리오의 와이어프레임·결정사항 표를 그대로 따른다
- **디자이너**: 와이어프레임을 기반으로 픽셀 디자인
- **QA·verifier**: 각 시나리오 E2E 테스트 케이스로 활용
- **새 모호함 발견 시**: 이 문서에 추가 + ADR 또는 LL 로 backfill

## 페르소나

```
주인공: 김 OO, 35세, 영업팀 대리, IT 친숙도 보통 (ChatGPT 가끔 씀)
배경:   사내 직원 ~300명 중 1명. 일일 평균 RAG 질의 ~20건.
운영:   회사 노트북 Chrome, 듀얼 모니터, 9~6 근무
```

# S1: 첫 로그인

## 무대
신규 직원이 admin 추가로 받은 임시 비밀번호 메일을 보고 첫 접속.

## 흐름
1. 메일 받음 → URL 클릭
2. 로그인 화면 (회원가입 버튼 X)
3. 임시 비밀번호 입력 → 변경 권장 토스트
4. 빈 채팅창 (Empty State)

## 결정사항

| 영역 | 결정 |
|------|------|
| 계정 부여 | admin 이 Panel 에서 사용자 추가 + 임시 비밀번호 메일 (SES). `ENABLE_SIGNUP=false` |
| 회원가입 버튼 | 표시 안 함 |
| 비밀번호 찾기 | 활성. SES 메일 자동 발송 |
| 첫 화면 | Open WebUI 기본 빈 채팅창 (내부 사용자 대상이라 튜토리얼 불필요) |
| 첫 로그인 비밀번호 변경 | 권장 토스트 + 마이페이지에서 변경 |
| 사이드 패널 디폴트 | 접힘 (사용자가 🎛 클릭 시 펼침) |
| 모델 변형 인지 | 상단 드롭다운 옆 한 줄 설명 |
| 메일 templating | 2종 (계정 발급 / 비밀번호 재설정), 한국어, 우리 브랜딩 |
| 사용 가이드 | `docs/user-guide.md` 1페이지, 계정 발급 메일에 링크 |
| 브랜딩 | Open WebUI 기본 + page title + favicon 만 변경 |
| 파일·이미지·URL 입력 | 클립 버튼 (Open WebUI 기본) + URL 자동 감지 (백엔드 정적 규칙) |
| 메타 질문 | 시스템 프롬프트에 "사용법·도움말" 패턴 → 정적 안내 답변 |

# S2: 단순 RAG 질문

## 무대
"A 상품 보증 기간이 얼마야?" 같은 단순 정의 질문.

## 흐름
1. 입력 → Enter
2. Open WebUI 기본 스피너 (~1.5~3초)
3. SSE 스트리밍 답변 + 출처 카드 5개
4. 후속 질문 또는 새 대화

## 결정사항

| 영역 | 결정 |
|------|------|
| 멀티라인 | Shift+Enter 줄바꿈, Enter 전송 (Open WebUI 기본) |
| 자동완성 | Phase 0 없음 |
| 로딩 인디케이터 | Open WebUI 기본 스피너. TTFT p95 ≤ 3초 메트릭 추적 |
| 취소 버튼 | Open WebUI 기본 Stop + SSE 취소 + audit partial 기록 |
| 마크다운 렌더 | **완성 후 한꺼번에 렌더** (ADR-0008 + N1 결정 — SSE 완성 후 PII 마스킹). 토큰별 흐름 X. 사용자는 스피너(M2-3) 표시 후 답변 일괄 도착 |
| 출처 카드 표시 개수 | Top-K 전부 (예: balanced=5개) |
| 출처 클릭 동작 | 청크 텍스트 100~200자 확장 표시 |
| 유사도 점수 | 표시 안 함 (사용자 이해 부담) |
| 원본 deep link | Phase 0 식별자만. Phase 1+ 고객사 deep link |
| LLM 인라인 인용 | 답변에 [1, 2] 표시 (LLM 이 실제 사용한 것만) |
| 좋아요 | 카운트만 저장 |
| 싫어요 | mini-form ("정확하지 않음 / 부족함 / 무관함 / 기타" + 텍스트) |
| 복사·재생성·공유 | Open WebUI 기본 (재생성=같은 설정, 공유=Phase 0 비활성) |
| 대화 컨텍스트 잘림 (10턴+) | Phase 0 조용히 자름. Phase 1+ toast |
| 자동 제목 | Open WebUI 기본 (첫 50자). Phase 1+ LLM 요약 |
| 대화 검색·삭제·보존 | Open WebUI 기본 + 영구 보관 (Phase 0) |

## 와이어프레임 (답변)
```
김OO ► A 상품 보증 기간이 얼마야?
```

# S3: SQL 질문

## 무대
"지난달 A 상품 매출 알려줘" 같은 수치 질문.

## 흐름
1. 의도 분류 LLM → SQL
2. SQL 생성 → 검증 → MySQL 실행 → 자연어화 LLM
3. 응답 ~7~8초

## 결정사항

| 영역 | 결정 |
|------|------|
| 응답 대기 인디케이터 | Open WebUI 기본 스피너 (M2-3 일관) |
| 실행 SQL 노출 | 디폴트 접힘 + "🔍 실행된 SQL 보기" 토글 |
| 의도 분류 결과 표시 | 출처 영역 작은 라벨 (📊 SQL / 📄 RAG / 🔀 종합 / 🌐 URL / 📎 파일 / 🖼 이미지) |
| SQL 출처 | "📊 데이터 출처: 회사 MySQL — transactions 테이블" 텍스트 |
| 다중 행 표시 | LLM 자연어화 그대로 (리스트·표 마크다운). 차트는 Phase 1+ |
| 빈 결과 | LLM 프롬프트에 "가능한 원인 1~2가지 짧게" 지시 |
| Validation 거부 | Generic 메시지 + 오류 ID (내부 로그에 정확 사유) |
| 1000행 cap | "결과가 너무 많아 상위 1000건만. 더 좁은 조건으로 재질문" 안내 |

## 와이어프레임
```
🤖 지난달(2026년 4월) A 상품 매출 총액은 **152,300,000원** 입니다.

   📊 SQL
   📊 데이터 출처: 회사 MySQL — transactions 테이블
   🔍 실행된 SQL 보기  ▶  (디폴트 접힘)

   👍 👎 🔁 📋
```

# S4: Hybrid 질문

## 무대
"보증 만료된 A 상품 고객 수와 보증 정책은?" — 수치 + 설명.

## 흐름
1. 의도 분류 → HYBRID
2. 병렬: SQL (~5초) + RAG (~2초)
3. RAG 결과부터 progressive 스트리밍
4. SQL 도착 후 종합 LLM → 완성

## 결정사항

| 영역 | 결정 |
|------|------|
| 응답 대기 | **부분 결과 progressive 표시** — RAG 답이 먼저 (TTFT ~2~3초), SQL 도착 후 보완. SSE 단계별 이벤트 |
| 출처 혼합 표시 | "🔀 종합" 라벨 + **두 영역 분리**: `📊 데이터 출처: ...` + `📎 문서 출처: [1] [2] ...` 카드 |
| 부분 실패 UX | 의도 라벨 변경 (SQL 실패=📄 RAG / RAG 실패=📊 SQL) + 답변 부분만 + [다시 시도] 버튼 |
| Hybrid Synthesis Style | Phase 0 사이드 패널만. Phase 1+ "수치 우선 / 문서 우선" 토글 |
| 출처 클릭 동작 일관성 | SQL 출처 = 펼침 토글 (M3-2) / RAG 청크 = 텍스트 확장 (M2-6). 같은 ▶ 아이콘 |

## 와이어프레임 (정상)
```
🤖 보증 만료된 A 상품 고객은 **12명** 입니다.

   A 상품의 보증 정책:
   - 보증 기간: 구매일로부터 2년
   - 정상 사용 조건 하 [1, 2]

   🔀 종합
   📊 데이터 출처: customers + transactions
   📎 문서 출처
   ┌──────────────────────────────────────┐
   │ [1] 계약서 #12345  ▶                  │
   │ [2] 약관 #67890    ▶                  │
   └──────────────────────────────────────┘
   🔍 실행된 SQL  ▶
```

## 와이어프레임 (SQL 실패 → 부분 답변)
```
🤖 A 상품 보증 정책:
   - 보증 기간: 2년 [1, 2]
   - ...

   📄 RAG (수치 정보 사용 불가)  [🔄 다시 시도]
   📎 문서 출처: ...
```

# S5: URL 첨부 질문

## 무대
"이 기사 요약해줘 https://news.example.com/article/123" — 외부 URL.

## 흐름
1. 정적 규칙 — URL 패턴 감지 → URL_FETCH
2. SSRF Guard → HTTP fetch (~1~5초) → readability4j 본문 추출
3. LLM 자연어화 → 응답

## 결정사항

| 영역 | 결정 |
|------|------|
| 다중 URL | 최대 3개까지 모두 fetch. 4개+ 면 "처음 3개만 처리" 안내. Rate Limit 카운트 N |
| URL 단독 | 디폴트 = 요약 + 핵심 3가지 (시스템 프롬프트) |
| SSRF 거부 | private IP → "사내 네트워크 주소입니다. 외부 공개 URL 만 분석 가능". 그 외 → generic + 오류 ID |
| 로딩 | M2-3 B 일관 (기본 스피너). `rag_url_fetch_latency_seconds` 메트릭 |
| URL 출처 카드 | 제목 + URL + 발행일 (썸네일 X, 본문 미리보기 X) |
| 본문 노출 | Phase 0 표시 X (답변만). Phase 1+ 토글 |
| 실패 응답 | 4종 일관: `⚠️ + 한 줄 + 오류 ID` (보안 위협 시) |
| Rate Limit 초과 | "URL 분석은 분당 10건. N초 후 다시" (Retry-After 헤더 활용) |

## 와이어프레임
```
🤖 이 기사는 ... 라고 보도합니다.
   핵심 내용:
   - 첫째, ...
   - 둘째, ...

   🌐 URL
   📎 URL 출처
   ┌─────────────────────────────────────────────┐
   │ 📰 기사 제목 (실제 페이지 제목)              │
   │ https://news.example.com/article/123        │
   │ 발행: 2026-05-15                             │
   └─────────────────────────────────────────────┘
```

# S6: 파일 첨부 질문

## 무대
8MB PPT (35슬라이드, 이미지 60개) 첨부 → "이 PPT 요약해줘".

## 흐름
1. 📎 클립 → OS 다이얼로그 → 첨부 미리보기 카드
2. Enter → 업로드 → Tika 추출 → Tesseract OCR 60개 → LLM → 응답
3. 총 ~60~80초 (동기 처리, RabbitMQ 미사용)

## 결정사항

| 영역 | 결정 |
|------|------|
| OS 다이얼로그 필터 | 화이트리스트 (`accept=".pdf,.docx,..."`) |
| 다중 첨부 미리보기 | 5개까지 가로 정렬. 누적 크기 (`8MB / 100MB`). 100MB 도달 시 거부 |
| 파일만 보내기 (질문 없음) | 디폴트 = "이 파일 요약해줘" 자동 (시스템 프롬프트). 빈 메시지 차단 |
| 업로드·처리 인디케이터 | **기본 스피너 (M2-3 일관)** — ⚠️ **LL 후보** (80초+ 정적 대기 위험. 운영 시 메트릭 모니터링) |
| 처리 중 취소 | Stop 버튼 → 서버 처리 중단 + S3 즉시 삭제 |
| 동시 작업 | 같은 대화 입력 차단 (Open WebUI 기본). 새 대화는 가능 |
| 파일 출처 카드 | 파일명·크기·페이지·OCR 이미지 수·처리 시간 + LLM 위치 인용 ("슬라이드 N") + 24h presigned URL |
| 후속 컨텍스트 유지 | **file_id 기억, 매 후속 DB 재로드**. 새 파일 첨부 시 기존 유지 (사용자 명시 제거 전까지) |
| 실패 메시지 | 5종 일관 (`⚠️ + 한 줄 + 권장 액션 + 오류 ID`) |
| 만료 안내 | 24h 후 file_id 만료 → "다시 업로드해주세요" |

## 와이어프레임 (응답)
```
🤖 이 PPT 의 핵심 내용:

   ## 1. 2026 Q2 사업 목표
   - 매출 150억 (전년 대비 +20%) (슬라이드 3)
   ...

   ## 2. 시장 동향 (슬라이드 12)
   - 차트: 분기별 매출 추이 우상향
   ...

   📎 파일
   📎 파일 출처
   ┌──────────────────────────────────────────────┐
   │ 회의자료_2026Q2.pptx (8.2MB, 35슬라이드)      │
   │ 📄 텍스트 추출 4,200 토큰                    │
   │ 🖼 이미지 OCR 60개                            │
```

# S7: 이미지 첨부 질문

## 무대
화이트보드 사진 (4096×3072, 3.2MB) 첨부 → "이 도표 설명해줘".

## 흐름
1. 📎 → 이미지 선택 → 썸네일 미리보기
2. Enter → VLM (qwen2.5-vl:7b) 호출 → 응답 ~3~5초

## 결정사항

| 영역 | 결정 |
|------|------|
| 클립 메뉴 | Open WebUI 기본 통합 (📎). 파일·이미지 분리 X |
| 미리보기 | Open WebUI 기본 썸네일 |
| 자동 리사이즈 (4096 초과) | 자동 + 카드에 작게 표시 (`4096×3072 로 자동 조정됨`) |
| 이미지 출처 카드 | **작은 썸네일 (120×120) + 메타 + 클릭 시 원본 모달 (라이트박스)** |
| VLM vs OCR 사용자 인지 | 의도된 분리. 사용자에게 명시 안 함 (채팅=VLM, 파일 안=OCR) |

| HEIC | Phase 0 거부 + "PNG/JPG로 변환 후 업로드" 안내. Phase 1+ libheif 자동 |
| PII 응답 마스킹 | **모든 LLM 응답 경로에 Layer 3 PII 마스킹 적용** — ADR-0008 신규 |

# S8: 검색 결과 없음

## 무대
"Z 상품 보증 기간" — 미등록 정보 / 또는 검색 1~2건만 (임계값 미달).

## 결정사항

| 영역 | 결정 |
|------|------|
| 진단 힌트 | 일반 응답 + 작은 진단 ("마지막 동기화: 30분 전 / 검색 모델: balanced") |
| 부분 결과 표시 | LLM 답변 + "임계값 미달" 라벨 + 청크 표시 (점수 비노출) |
| Actionable Empty State | **3가지 행동 버튼** — `🔍 광범위 검색 (broad 자동 재실행)` / `📝 다른 표현으로 재질문` / `🚩 신고` |
| broad 자동 재실행 | "🔍 광범위 검색" 버튼 = 같은 질문 + model=broad 자동 호출 |
| 반복 실패 학습 | Phase 0 미명시. Phase 1+ 데이터 카탈로그 + admin 대시보드 |
| 신고 채널 | "🚩 신고" → `audit_log(action=user_report_missing_data)` + Phase 1+ admin 다이제스트 |

## 와이어프레임 (0건)
```

   다음 중 선택:
   [🔍 광범위 검색] [📝 다른 표현으로 재질문] [🚩 신고]
```
