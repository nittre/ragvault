# 위젯 서비스 어드민 매뉴얼

**위젯 서비스**(`app-widget`)의 관리자 콘솔(Widget Admin, `/admin`, 로컬 포트 `5173`) 사용법입니다. 외부 고객사 웹사이트에 `<script>` 한 줄로 삽입되는 임베드 챗봇을 운영하기 위한 콘솔로, 챗 서비스 콘솔과는 **별개의 애플리케이션·별개의 DB(`widget_db`)**를 사용합니다.

> 챗 서비스 어드민 콘솔과 화면 구성·동작이 **동일한 항목은 이 문서에서 반복하지 않고 [챗 서비스 어드민 매뉴얼](admin-manual.md) 참고로 대체**합니다. 이 문서는 위젯 서비스에서 **다르거나 위젯에만 있는** 부분 위주로 설명합니다.

---

## 1. 로그인 / 사용자 관리 / 비밀번호 변경 — 챗 서비스와 동일

로그인 화면(이메일·비밀번호, 초기 비밀번호 변경 강제), 사용자 관리(`/admin/users`, `SUPER_ADMIN`만 생성·역할변경·비밀번호 재설정·삭제 가능), 사이드바 하단의 비밀번호 변경(`/admin/settings`)은 챗 서비스와 화면·동작이 사실상 동일합니다. **[어드민 매뉴얼 §2, §16] 참고.**

다만 역할 종류가 다릅니다 — 위젯 어드민에는 일반 `USER` 역할이 없고 **`ADMIN` / `SUPER_ADMIN`** 두 가지만 존재합니다(위젯은 방문자용 로그인이 없는 운영자 전용 콘솔이기 때문).

---

## 2. 화면 구조

좌측 사이드바는 상단 로고 아래 "Widget Admin" 배지가 표시되고, 메뉴는 **관리**(정적 메뉴)와 **데이터소스**(등록한 고객 DB별) 두 섹션으로 나뉩니다.

**관리**: 사용자 관리 · 지식문서 · 대화 로그 · 사용량 통계 · PII 마스킹 · 감사 로그 · 검색 설정 · **위젯 설정**(Site-Key) · 쿼리 콘솔

**데이터소스**: 데이터소스 목록으로 이동하는 "관리" 링크 + 등록된 각 데이터소스를 펼치면 RAG 테이블 / SQL 테이블 하위 메뉴가 나타납니다.

---

## 3. 지식문서 관리 (`/admin/knowledge`)

업로드 형식(`.md .docx .xlsx .pptx .pdf`), 파일 업로드·마크다운 추가·편집·재임베딩·전체 재임베딩·삭제 등 화면 구성과 동작이 챗 서비스와 **완전히 동일**합니다. **[어드민 매뉴얼 §3] 참고.**

---

## 4. 대화 로그 (`/admin/conversations`) — 위젯 전용

방문자와 위젯 챗봇 간의 실제 대화 이력을 조회합니다.

- **사이트키로 필터** 입력창으로 특정 Site-Key의 대화만 조회 가능 (20건/페이지, 이전/다음 페이지 이동) — **부분 일치·대소문자 구분 없이** 검색되므로 Site-Key 전체를 정확히 입력하지 않아도 일부 문자열만으로 조회할 수 있습니다.
- 목록 컬럼: 시간·사이트키·사용자 질문(100자 축약)·봇 응답(100자 축약)·**문서매칭**(✓/✗ — 내부 지식문서에서 근거를 찾았는지)·**차단**(프롬프트 인젝션 등으로 응답이 차단된 경우만 배지 표시)
- 행을 클릭하면 **대화 상세** 모달이 열려 질문·응답 전문, 문서 매칭 여부, 매칭된 소스 수를 확인할 수 있습니다.

---

## 5. 사용량 통계 (`/admin/stats`)

챗 서비스의 사용량 통계(`/admin/usage-stats`)와 동일한 형태이며, 모든 지표가 최근 30일 고정 집계입니다.

- 요약 카드 4개: **전체 질의**(누적) · **최근 7일** 질의 수 · **최근 30일 문서 매칭율**(%) · **최근 30일 차단율**(%)
- **일별 질의 추이(최근 30일)** 막대 그래프 — 막대에 마우스를 올리면 해당 날짜의 건수가 툴팁으로 표시됩니다.
- **라우팅 상세** — 최근 30일 요청을 RAG/SQL/HYBRID/차단/기타 경로로 분류해 가로 막대 그래프와 표(건수·비율)로 표시
- **실행 통계** — SQL이 실제로 실행된 횟수(HYBRID 요청 내부에서 실행된 것 포함). 위젯은 웹 검색 라우팅을 지원하지 않아 SQL 실행 횟수만 표시됩니다.

---

## 6. PII 마스킹 규칙 (`/admin/masking`) — 챗 서비스와 다름

챗 서비스는 데이터소스별 규칙 + 레벨(기본/강화) + PII 자동 탐색 기능이 있지만, 위젯은 **전역 단일 목록**이며 더 단순합니다.

- **규칙 추가** — 이름, 패턴(정규식), 교체값, **순서**(숫자, 검증 시 적용 순서), 활성화 체크박스를 입력해 생성
- 목록 컬럼: 순서·이름·패턴·교체값·활성화(배지)·편집/삭제
- **캐시 재로드** 버튼 — 규칙 변경 후 런타임 캐시를 즉시 갱신
- PII 자동 탐색 기능, standard/aggressive 레벨 구분, 데이터소스별 스코프 개념은 **위젯에는 없습니다.**

---

## 7. 검색 설정 (`/admin/search`) — 챗 서비스와 다름

챗 서비스의 범용 키-값 편집 테이블과 달리, 위젯은 **의미별 카드**로 구성되어 있습니다.

- **검색 결과 수 (top_k)** — 숫자 입력(1~20), 저장 버튼
- **유사도 임계값 (threshold)** — 슬라이더(0~1) + 숫자 입력 동시 조정
- **결과 없음 응답 메시지 (`no_results_response`)**, **주입 차단 응답 메시지 (`injection_blocked_response`)** — 각각 textarea로 문구 직접 수정
- **위젯 SQL 조회 허용 (`sql_enabled`)** — 토글 스위치. 화면에 "기본값은 비활성이며, 본인 인증 흐름이 없는 상태에서는 임의 데이터 조회 위험이 있으므로 **내부 검증·데모 용도로만 사용**하세요"라는 경고 문구가 표시됩니다. **운영 환경에서 방문자 인증 없이 이 옵션을 켜는 것은 데이터 노출 위험이 있으므로 신중히 판단해야 합니다.**
- 위 항목 외에 알 수 없는 설정 키가 있으면 하단에 텍스트 카드로 자동 표시됩니다.

---

## 8. 감사 로그 (`/admin/audit`) — 챗 서비스와 거의 동일

시각·행위자·액션·대상 유형·대상 ID 컬럼은 동일하며, 위젯 쪽에는 **IP 주소** 컬럼이 추가로 있고 행을 클릭하면 상세 모달(행위자, 액션 배지, 대상, IP, 상세 JSON)이 열립니다. 페이지네이션은 30건/페이지입니다. **[어드민 매뉴얼 §4] 참고, 차이점만 위 내용 확인.**

---

## 9. 위젯 설정 — Site-Key 관리 (`/admin/widget`) — 위젯 전용

임베드 스니펫이 사용할 **Site-Key**와 위젯의 표시 방식(브랜딩)을 카드 그리드로 관리합니다.

- **새 site-key 발급** 모달 입력 항목: **레이블**(필수, 내부 구분용), **봇 이름**(위젯 채팅창 상단에 표시), **인사말**(첫 화면 문구), **브랜드 색상**(컬러 피커), **로고 URL**
- 카드에는 브랜드 색상 스와치, 레이블, 활성/비활성 배지, site-key 값(클릭 또는 복사 아이콘으로 클립보드 복사), 봇 이름, 발급일이 표시됩니다.
- 카드 하단 액션: **편집**(레이블·봇이름·인사말·브랜드색상·로고URL·활성화 여부 수정), **활성화/비활성화 토글**, **삭제**(확인창: "삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다")
- 이 Site-Key 값이 고객사 웹사이트에 심는 `<script>` 스니펫의 `X-Site-Key` 인증에 사용됩니다.

---

## 10. 쿼리 콘솔 (`/admin/query-console`) — 위젯 전용

관리자가 운영 중인 text-to-sql 파이프라인을 직접 테스트해볼 수 있는 화면입니다.

- 자연어 질문을 입력창에 입력 후 **실행** — LLM이 생성한 **SQL**(코드 블록)과 **의도(intent)**, 최종 **응답** 텍스트를 그대로 확인할 수 있습니다.
- 실제 방문자 채팅과 동일한 파이프라인을 관리자 권한으로 미리 검증하는 용도입니다 (SQL 검증·화이트리스트·민감도 규칙은 실제 채팅과 동일하게 적용됨).

---

## 11. 데이터소스 관리 (`/admin/datasources`) — 챗 서비스와 다름

고객사 소유 DB를 등록하는 화면으로, **SSH 터널 기능이 없고** 더 단순한 폼을 가지며, 대신 **테이블 관리 인라인 패널**이 추가되어 있습니다.

### 등록 폼
이름, 설명, DB 타입(MySQL/MariaDB), 호스트, 포트, 데이터베이스명, 사용자명, 비밀번호만 입력합니다. (SSH 터널·Bastion·PEM 키·자동 설명 생성 체크박스는 위젯에 없습니다.)

### 목록 화면
- 컬럼: 이름/설명 · DB 타입 배지 · `호스트:포트 / DB명` · 활성 상태
- **연결 테스트** — 결과가 인라인으로 초록/빨강 아이콘과 메시지로 표시
- **RAG 테이블** / **SQL 테이블** 바로가기 링크
- **삭제**(확인창: "삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다")
- **테이블 관리** 버튼 — 클릭 시 행 아래로 해당 데이터소스의 **전체 테이블 목록**이 펼쳐지는 인라인 패널이 열립니다.

### 테이블 관리 패널 (위젯 고유 기능)
- 소스 DB의 모든 테이블을 테이블명·설명·마지막 동기화 시각과 함께 나열
- RAG 미등록 테이블: **RAG 추가** 버튼으로 즉시 등록
- RAG 등록된 테이블: **동기화**(해당 테이블만 즉시 재동기화 실행) / **RAG 제거** 버튼
- 각 테이블 아래에 최근 동기화 작업 이력(최대 3건)이 상태 배지(대기중/실행중/완료/실패)·처리 행수·시작 시각·오류 메시지와 함께 표시됩니다.

> 챗 서비스처럼 데이터소스 단위의 "자동 동기화 ON/OFF" 토글은 없으며, 테이블 단위로 그때그때 **동기화** 버튼을 눌러 처리하는 방식입니다.

---

## 12. RAG 테이블 / SQL 테이블 (`/admin/datasources/:dsId/rag-tables`, `/sql-tables`)

화면 구성·컬럼·모달(스키마 탐색, 컬럼 설정, 테이블·컬럼 설명 편집, 민감도 4단계, 일괄 삭제/등록)이 챗 서비스와 **거의 동일**합니다. **[어드민 매뉴얼 §10, §11] 참고.**

차이점만 정리하면:
- 상단에 "← 데이터소스로 돌아가기" 링크가 추가로 있습니다.
- RAG 테이블 화면에 챗 서비스에 있는 "자동 동기화 ON/OFF" 토글과 "지금 동기화" 버튼이 **없습니다** — 개별 테이블 동기화는 위 §11의 데이터소스 "테이블 관리" 패널에서 수행합니다.
- SQL 테이블 화면도 "자동 동기화" 토글이 없고, **드리프트 확인**·**캐시 갱신**·**스키마 탐색**·**직접 추가**만 제공됩니다.

---

## 13. 운영상 주의사항

- **Site-Key는 고객사 웹사이트에 노출되는 공개 값**입니다(JWT나 API 키처럼 비밀로 취급되지 않음). 다만 비활성화·삭제하면 해당 스니펫이 즉시 동작을 멈추므로, 이미 배포된 위젯의 Site-Key를 삭제할 때는 고객사에 사전 공지가 필요합니다.
- **위젯 SQL 조회 허용(`sql_enabled`)**은 기본 비활성이며, 방문자 인증 체계가 없는 공개 위젯 특성상 켜면 임의 데이터 조회로 이어질 수 있으니 운영 환경에서는 원칙적으로 끄고, 필요 시 내부 검증·데모 용도로만 한시적으로 사용하세요.
- 위젯 서비스는 **사내 챗 서비스 데이터에 접근할 수 있는 경로 자체가 없습니다**(별도 앱 + 별도 DB `widget_db`). 데이터소스는 반드시 고객사 소유 DB만 등록해야 합니다.
- 사용자 삭제·비밀번호 재설정은 챗 서비스와 마찬가지로 `SUPER_ADMIN`만 가능하며 확인 절차가 간단하므로 대상 이메일을 재차 확인 후 실행하세요.
