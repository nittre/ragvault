# 챗 서비스 어드민 매뉴얼

**챗 서비스**(`app-internal`)의 관리자 콘솔(`/admin`) 사용법입니다. 좌측 사이드바에서 채팅 화면의 **관리자 설정** 메뉴로 진입할 수 있으며, 계정 역할이 `ADMIN` 또는 `SUPER_ADMIN`이어야 접근할 수 있습니다(그 외 역할은 403 화면이 표시됩니다).

> 일반 채팅 화면 사용법은 [사용자 매뉴얼](user-manual.md)을 참고하세요.

---

## 1. 화면 구조

좌측 메뉴는 **전역** 항목과 **데이터소스**별 항목으로 나뉩니다.

**전역** (데이터소스와 무관하게 서비스 전체에 적용)
- 사용자 관리 · 지식문서 · 감사 로그 · SQL 실행 로그 · 사용량 통계 · API 키 · 파라미터 한도

**데이터소스** (등록한 외부 DB마다 개별 설정)
- DS 관리(데이터소스 목록) 및 각 데이터소스 하위에 RAG 테이블 · SQL 테이블 · 백과사전 · PII 마스킹 · DDL 이벤트 · 검색 설정

사이드바 하단에서 비밀번호 변경, 채팅 화면으로 돌아가기가 가능합니다.

### 권한 — ADMIN vs SUPER_ADMIN

사용자 관리 화면의 **사용자 추가, 역할 변경, 비밀번호 재설정, 삭제** 기능은 `SUPER_ADMIN`만 사용할 수 있습니다. 일반 `ADMIN`은 사용자 목록 조회만 가능합니다.

---

## 2. 사용자 관리 (`/admin/users`)

- **사용자 추가**(SUPER_ADMIN 전용) 버튼으로 이메일·이름·초기 비밀번호(8자 이상)·역할(`USER`/`ADMIN`/`SUPER_ADMIN`)을 입력해 계정을 생성합니다.
- 목록 테이블에서 바로 수정 가능한 항목(SUPER_ADMIN 한정):
  - **역할** — 드롭다운으로 즉시 변경
  - **상태** — 활성/비활성 배지를 클릭하면 토글
- **비밀번호 재설정** — 대상 사용자 행의 링크를 누르면 모달이 열리고, 새 비밀번호/확인을 입력해 즉시 재설정합니다(사용자 본인 확인 절차 없이 관리자가 바로 설정).
- **삭제** — 클릭 시 확인창(`이메일 을(를) 삭제하시겠습니까?`) 후 삭제됩니다.

---

## 3. 지식문서 관리 (`/admin/knowledge`)

RAG가 검색하는 지식문서(전역, 데이터소스 무관)를 관리합니다.

- 지원 형식: `.md · .docx · .xlsx · .pptx · .pdf` (이미지·표 포함 문서도 벡터화됨)
- **파일 업로드** — 위 형식의 실제 파일을 업로드
- **마크다운 추가** — 파일명과 마크다운 본문을 직접 입력해 문서 생성
- 텍스트 파일(`.md/.txt` 등)은 목록에서 **편집** 버튼으로 내용을 바로 수정 가능. 그 외 형식(docx/pdf 등)은 편집 불가, 재업로드 필요.
- **재임베딩** — 개별 문서를 다시 벡터화. **전체 재임베딩** — 전체 문서 일괄 재처리.
- **삭제** — 즉시 삭제(확인창 있음).
- 목록에는 파일명·형식(확장자 배지)·크기·수정일이 표시됩니다.

---

## 4. 감사 로그 (`/admin/audit-logs`)

관리자·시스템 작업 이력을 페이지네이션(50건/페이지) 테이블로 조회합니다. 컬럼: 시각·행위자·액션·대상 유형·대상 ID·상세.

---

## 5. SQL 실행 로그 (`/admin/sql-logs`)

LLM이 생성한 SQL의 검증·실행 이력을 조회합니다(최대 200건, 20건/페이지).

- 카테고리 필터: 전체 / 환각 컬럼 / NULL·0 나눗셈 / 풀스캔 / 구문 오류 / 집계 오류 / 기타
- 컬럼: 시각·사용자·질문·생성 SQL(클릭하면 펼침/접기)·카테고리·검증(허용/거부)·실행 결과(성공/오류/타임아웃)·소요시간(ms)·사유 또는 오류 메시지

---

## 6. 사용량 통계 (`/admin/usage-stats`)

위젯 서비스 어드민(`/admin/stats`)과 동일하게 모든 지표가 **최근 30일 고정 집계**입니다 (날짜 선택기 없음).

- 요약 카드 4개: **전체**(누적) · **최근 7일** 건수 · **최근 30일 문서 매칭율**(%) · **최근 30일 차단율**(%)
- **일별 질의 추이(최근 30일)** 막대 그래프
- **라우팅 상세** — 최근 30일 요청을 RAG/SQL/파일 업로드/HYBRID/웹 검색/차단/기타 경로로 분류해 가로 막대 그래프와 표(건수·비율)로 표시
- **실행 통계** — SQL/웹 검색이 실제로 실행된 횟수(HYBRID 요청 내부에서 실행된 것 포함). 라우팅 분류와 합계가 다를 수 있습니다.

---

## 7. API 키 관리 (`/admin/api-keys`)

외부 API 클라이언트가 서버 간 호출 시 사용할 인증 키를 관리합니다.

- **키 발급** — 키 이름, 스코프(예: `chat`, `chat,admin` — 콤마 구분 직접 입력), 만료일(선택)을 입력해 발급
- 발급 직후 실제 키 값이 노란 배너로 **한 번만** 표시됩니다. 👁 아이콘으로 보기/숨기기, 복사 아이콘으로 클립보드 복사 가능 — **다시 조회할 수 없으므로 반드시 이 시점에 복사해야 합니다.**
- **교체(Rotate)** — 확인 후 기존 키를 즉시 폐기하고 새 키를 발급(마찬가지로 그 순간만 표시)
- **폐기(삭제)** — 확인 후 즉시 비활성화
- 목록 컬럼: 이름·키 접두사(일부만 표시)·스코프·만료일·최근 사용일·상태(활성/폐기됨)

---

## 8. 파라미터 한도 (`/admin/param-limits`)

사용자 채팅 화면의 파라미터 패널에서 조정 가능한 각 파라미터의 **허용 범위**를 관리합니다.

- 각 파라미터 행에서 **편집** → 최솟값/최댓값 입력 → 저장
- **잠금(🔒)** — 사유를 입력하면 해당 파라미터를 사용자가 아예 조정하지 못하도록 잠금. 잠긴 항목은 행 배경이 주황색으로 표시되고 사유가 노출됨
- **잠금 해제** — 확인 후 해제

---

## 9. 데이터소스 관리 (`/admin/ds`)

Text-to-SQL·RAG의 대상이 되는 외부 MySQL/MariaDB를 등록·관리합니다.

### 등록/수정 폼 주요 항목
- **이름**, **설명**(LLM이 데이터소스를 선택할 때 참고하는 라우팅 힌트)
- **테이블·컬럼 설명 자동 생성** 체크박스(신규 등록 시만) — 등록 후 LLM이 각 테이블·컬럼 의미를 자동으로 채움(DB `COMMENT`가 있으면 그것을 우선 사용)
- DB 종류(MySQL/MariaDB), 호스트, 포트, 데이터베이스명, 사용자명, 비밀번호(수정 시 변경할 때만 입력, 비워두면 기존 값 유지)
- **SSH 터널 사용** 체크 시 Bastion 호스트/포트, SSH 사용자명, **PEM 개인키**(전체 내용을 그대로 붙여넣기, 서버에 암호화 저장되며 이후 재조회되지 않음), Passphrase(선택) 입력

### 목록 화면
- DB 종류 배지, 이름, SSH 사용 여부, `호스트:포트/DB명`, 활성 상태
- **테스트** — 연결 테스트 실행 후 연결 OK/실패 표시
- **수정** / **삭제**(확인창 있음)
- 펼치기(▼)로 사용자명·등록일·수정일·(SSH 사용 시) 터널 경로(`user@host:port → 실DB호스트:포트`) 확인

각 데이터소스를 클릭하면 좌측 메뉴에 하위 항목(RAG 테이블/SQL 테이블/백과사전/PII 마스킹/DDL 이벤트/검색 설정)이 펼쳐집니다.

---

## 10. RAG 테이블 (`/admin/ds/:dsId/rag-tables`)

벡터 임베딩 대상이 되는 테이블을 관리합니다.

- **자동 동기화 ON/OFF** 토글 — MySQL binlog 기반 실시간 동기화 여부. OFF에서 다시 ON으로 켤 때, 비활성화 시점 이후의 변경사항을 즉시 동기화할지 확인창이 뜹니다.
- **변경사항 확인**(자동 동기화 OFF일 때만 노출) — 소스 DB와의 드리프트(테이블 삭제·컬럼 변경)를 확인. 테이블명 옆에 `⚠ 테이블 없음` / `⚠ 컬럼 변경` 경고로 표시됩니다.
- **지금 동기화** — binlog를 즉시 동기화 실행, 결과(성공/실패 건수) 배너 표시
- **스키마 탐색** — 소스 DB의 테이블 목록을 조회해 여러 테이블을 체크박스로 선택 후 일괄 등록(이미 등록된 테이블은 선택 불가, "등록됨" 배지)
- **직접 등록** — 테이블명을 수동 입력해 단건 등록
- 테이블 목록 컬럼: 테이블명(드리프트 경고 포함)·PK·Title 컬럼·Content 컬럼(임베딩 대상, LLM 자동 분류 중이면 "⟳ 분류중" 표시, 미설정이면 "⚠ 미설정")·청킹 전략/크기·활성 토글·작업
- 행별 작업: **컬럼 설정**(임베딩할 PK/Title/Content/Metadata 컬럼과 청킹 전략·청크 크기·오버랩을 설정하는 모달, 컬럼 미설정 시 텍스트 계열 컬럼을 자동 추천), **임베딩 자동설정**(컬럼 설정을 LLM으로 재분석 후 전체 재임베딩), **삭제**
- 여러 행을 체크하면 상단에 **일괄 자동설정** / **일괄 삭제** 액션 바가 나타납니다.

## 11. SQL 테이블 화이트리스트 (`/admin/ds/:dsId/sql-tables`)

Text-to-SQL이 조회할 수 있는 테이블을 화이트리스트 방식으로 관리합니다(등록하지 않은 테이블은 SQL 생성 대상에서 제외).

- **자동 동기화 ON/OFF** — RAG 테이블과 동일한 개념. OFF→ON 전환 시 그동안의 변경분을 재생(replay)할지 확인 후 적용 결과(적용/건너뜀 건수) 표시
- **드리프트 확인**, **지금 동기화**, **캐시 갱신**(스키마 캐시 새로고침), **스키마 탐색**(일괄 등록), **직접 추가**(테이블명 + 표시명)
- 목록 컬럼: 테이블명(드리프트 경고 포함)·표시명·**설명**(클릭하면 테이블/컬럼 설명 편집 모달)·**민감도**(`public`/`internal`/`confidential`/`restricted`, 드롭다운으로 즉시 변경, LLM 분류 대기 중이면 "⟳ 분류중")·활성 토글·삭제
- **테이블·컬럼 설명 편집 모달** — 테이블 설명과 컬럼별 설명을 입력. 이 설명은 SQL 생성·데이터소스 라우팅 프롬프트에 그대로 주입됩니다. *계산식·필터·여러 테이블을 넘나드는 규칙은 여기가 아니라 "백과사전"에 작성해야 합니다.*
- 여러 행 체크 시 **일괄 삭제** 가능

---

## 12. 백과사전 (`/admin/ds/:dsId/knowledge`)

데이터소스별 비즈니스 규칙·계산식·도메인 도출 쿼리를 항목 단위로 관리합니다.

- **항목 추가** → 모달에서 제목, 역할(**규칙**: 정의·공식·제약 / **측정**: 도메인 도출 쿼리 — SQL 생성 시 few-shot 예시로 LLM에 전달됨), 본문, **고정**(체크 시 항상 프롬프트에 주입) 입력
- 목록에서 **수정**(연필 아이콘)/**삭제**(휴지통 아이콘, 확인창 없이 즉시 저장되므로 주의)

---

## 13. PII 마스킹 규칙 (`/admin/ds/:dsId/masking-rules`)

LLM 응답에 포함될 수 있는 개인정보를 정규식으로 마스킹하는 규칙을 관리합니다.

- **규칙 직접 추가** — 규칙 이름, 정규식 패턴(예: `01[016789]-?\d{3,4}-?\d{4}`), 치환 토큰(예: `[전화번호]`), 레벨(기본/강화), 적용 순서(숫자, 작을수록 먼저 적용)
- **PII 자동 탐색** — 스키마 컬럼명을 분석해 PII로 의심되는 컬럼을 자동 탐지, 유형별로 체크해 **일괄 생성** 가능. 각 항목을 펼치면 감지된 컬럼 목록·치환 토큰·레벨 확인 가능
- 목록에서 인라인 **편집**, **활성/비활성 토글**(초록 스위치), **삭제**(확인창 있음)
- 스코프 컬럼에 **전역**(모든 데이터소스 공통) / **DS 전용** 구분 표시

---

## 14. DDL 이벤트 (`/admin/ds/:dsId/ddl-events`)

MySQL 소스 DB의 스키마 변경(DDL) 이력을 조회하는 읽기 전용 화면입니다. 이벤트가 없으면 초록 체크 아이콘과 함께 "DDL 이벤트가 없습니다" 표시. 각 이벤트에 테이블명·이벤트 타입·위험도(HIGH/MEDIUM/LOW, 색상 배지)·실제 SQL 쿼리·발생 시각이 표시됩니다.

---

## 15. 검색 설정 (`/admin/ds/:dsId/search-config`)

RAG 검색 동작을 제어하는 키-값 설정을 관리하는 화면입니다(예: 웹검색 활성화 여부 등 `rag.*` 계열 설정). 값 셀을 클릭하거나 **편집**을 눌러 값을 수정하고 Enter로 저장, Esc로 취소합니다.

---

## 16. 운영상 주의사항

- **API 키 원문은 발급/교체 직후 한 번만 노출됩니다.** 그 화면을 벗어나면 다시 볼 수 없으므로 즉시 복사해야 합니다.
- **SSH PEM 개인키·Passphrase는 서버에 암호화 저장되며, 수정 화면에서도 기존 값이 내려오지 않습니다.** 값을 바꾸지 않으려면 해당 입력란을 비워두면 됩니다.
- 사용자 **삭제**, **비밀번호 재설정**은 `SUPER_ADMIN`만 가능하며 별도 확인 절차 없이 즉시 적용되는 항목(재설정)도 있으니 대상 이메일을 재차 확인 후 실행하세요.
- 자동 동기화를 껐다가 다시 켤 때는 그 사이 발생한 변경분을 즉시 반영할지 여부를 반드시 확인창에서 판단해야 합니다(대량 재동기화가 발생할 수 있음).
- SQL 테이블의 **민감도(`data_sensitivity`)**는 `public/internal/confidential/restricted` 4단계이며, 이 값에 따라 SQL 생성·조회 허용 범위가 서버 쪽에서 통제됩니다.
